# 創意思考與創業 SOP

---

## 一、找靈感（Idea Generation）

### 1. 問題導向法（Problem-First）

- 觀察自己和身邊人每天遇到的**痛點**，記錄下來
- 問：「這件事為什麼這麼麻煩？」「有沒有更好的方式？」
- 最好的產品不是發明新需求，而是**解決已存在的問題**
- 框架：「我希望有一個 _____ 能幫我 _____」

### 2. 靈感來源清單

| 來源 | 做法 |
|---|---|
| 日常不便 | 記錄每天讓你皺眉的 3 件事 |
| 產業趨勢 | 讀 TechCrunch、Product Hunt、Hacker News |
| 既有產品缺陷 | 看 App Store 一星評論，找使用者真正的痛 |
| 跨領域移植 | A 產業的解法搬到 B 產業（Uber 模式 → 各行各業） |
| 個人專長交叉 | 你的技能 × 你的興趣 × 市場需求 的交集 |
| 社群與論壇 | Reddit、PTT、Dcard 上人們在抱怨什麼 |
| 政策與法規變動 | 新法規 = 新需求（如：GDPR → 隱私工具爆發） |

### 3. 創意發散技巧

- **腦力激盪（Brainstorming）**：先求量不求質，禁止批評，10 分鐘寫 20 個點子
- **心智圖（Mind Map）**：從核心問題向外延伸關聯概念
- **SCAMPER 法**：替代（Substitute）、結合（Combine）、調整（Adapt）、修改（Modify）、挪用（Put to other use）、消除（Eliminate）、反轉（Reverse）
- **逆向思考**：「怎樣讓這件事變得更糟？」→ 反過來就是改善方向
- **類比思考**：「如果 Netflix 來做這件事會怎麼做？」
- **限制激發創意**：刻意加約束（只用 100 元、只有一頁、24 小時內完成）

### 4. 靈感捕捉系統

- 隨時記錄：手機備忘錄建一個「Ideas」資料夾
- 每週回顧一次，標記有潛力的點子
- 不要評判剛冒出來的想法，先記下來再說
- 洗澡、散步、睡前是靈感高峰期 → 手邊備好記錄工具

## 二、驗證想法（Idea Validation）

### 1. 驗證漏斗（由快到慢）

```
想法 → Google 搜尋是否有人做過
     → 找 5 個目標用戶聊聊
     → 做最小原型測試
     → 看有沒有人願意付錢
```

### 2. 用戶訪談（Customer Interview）

- 找 **5-10 個潛在用戶**，不是朋友家人（他們會客氣）
- 問過去的行為，不問未來意願：
  - ✅「你上次遇到這個問題時怎麼解決的？」
  - ❌「如果有這個產品你會用嗎？」（所有人都會說會）
- 聽他們怎麼**描述問題**，用他們的語言做行銷
- 推薦閱讀：《The Mom Test》

### 3. 競品分析

- 有競爭者是**好事**，代表市場存在
- 分析：他們的定價？目標客群？一星評論寫什麼？
- 找到差異化：更便宜、更快、更專注某個族群、更好的體驗
- 工具：SimilarWeb、Crunchbase、App Annie

### 4. 付費意願測試

- 問：「如果這個產品存在，你願意付多少錢？」
- 更好的方式：做一個預購頁面，看有沒有人真的刷卡
- 最強驗證 = 有人在產品還沒做出來之前就願意付錢

## 三、MVP（最小可行產品）

### 1. MVP 核心原則

- 只做**一個核心功能**，把它做到能用
- 目標不是完美，是**學習**——用最低成本驗證假設
- 能用 No-Code 就不寫 Code，能手動就不自動化
- 時間限制：**2-4 週**內必須上線第一版

### 2. MVP 類型

| 類型 | 說明 | 適用情境 |
|---|---|---|
| Landing Page | 一頁式網站 + 註冊按鈕 | 測試需求是否存在 |
| 影片 MVP | 錄一段 demo 影片展示概念 | Dropbox 就是這樣起步的 |
| 手動 MVP（Wizard of Oz） | 前台看起來自動，後台人工處理 | 驗證流程可行性 |
| 拼裝 MVP（Concierge） | 用現有工具拼湊（Notion + Zapier + Google Form） | 快速測試完整流程 |
| 單一功能 App | 只做核心功能的精簡版 | 技術產品 |

### 3. MVP Checklist

- [ ] 核心問題是什麼？（一句話描述）
- [ ] 核心功能是什麼？（只選一個）
- [ ] 目標用戶是誰？（越具體越好）
- [ ] 成功指標是什麼？（例：100 人註冊、10 人付費）
- [ ] 預計多久做完？（超過 4 週就砍功能）

## 四、商業模式（Business Model）

### 1. 商業模式畫布（Business Model Canvas）

九大區塊一頁搞定：

| 區塊 | 問題 |
|---|---|
| **價值主張** | 你解決什麼問題？為什麼選你？ |
| **目標客群** | 誰最需要這個？ |
| **通路** | 怎麼接觸到客戶？ |
| **客戶關係** | 如何留住客戶？ |
| **收入來源** | 怎麼賺錢？ |
| **關鍵資源** | 需要什麼才能運作？ |
| **關鍵活動** | 每天必須做什麼？ |
| **關鍵夥伴** | 需要誰的幫助？ |
| **成本結構** | 主要花費是什麼？ |

### 2. 常見獲利模式

- **訂閱制（SaaS）**：月費 / 年費，穩定現金流
- **免費增值（Freemium）**：基本免費，進階付費
- **交易抽成（Marketplace）**：每筆交易抽 5-20%
- **廣告模式**：流量大才適用，門檻高
- **一次性付費**：簡單但缺乏持續收入
- **授權 / API**：賣技術給其他公司

### 3. 定價策略

- 不要從成本定價，從**客戶感受到的價值**定價
- 提供 3 個方案（錨定效應）：基本 / 專業 / 企業
- 初期可以低價獲客，但不要免費（免費用戶不珍惜）
- 每 6 個月重新評估定價

## 五、產品開發流程（Build → Measure → Learn）

### 1. 精實創業循環（Lean Startup Loop）

```
想法 → 建造（Build）→ 產品
產品 → 測量（Measure）→ 數據
數據 → 學習（Learn）→ 新想法
```

- 每個循環越短越好，目標是**快速學習**
- 不要一次做太多功能，每次只測試一個假設

### 2. 用戶回饋收集

- 上線第一天就開始收集回饋
- 管道：App 內回饋按鈕、Email、社群、用戶訪談
- 區分「想要」和「需要」：用戶說想要的不一定是真正需要的
- 看用戶**怎麼用**，不只聽他們**怎麼說**

### 3. 關鍵指標（Metrics That Matter）

| 階段 | 關鍵指標 |
|---|---|
| 驗證期 | 註冊轉換率、付費意願 |
| 成長期 | DAU/MAU、留存率（Day 1 / Day 7 / Day 30） |
| 獲利期 | LTV（客戶終身價值）、CAC（獲客成本）、MRR（月經常性收入） |

- **北極星指標**：選一個最能代表產品價值的指標，全公司對齊
- LTV > 3× CAC 才是健康的商業模式

### 4. Pivot（轉向）時機

- 連續 3 個月核心指標沒成長
- 用戶用你的產品做你沒預期的事（可能是更好的方向）
- 市場環境改變（新競爭者、政策變動）
- Pivot ≠ 失敗，是用學到的東西走更對的路

## 六、團隊與執行（Team & Execution）

### 1. 共同創辦人

- 互補 > 相似：一個做產品，一個做技術，一個做商務
- 在一起工作過再合夥（先做 side project 測試默契）
- 提前談好：股權分配、決策方式、退出機制
- 股權建議：Vesting 4 年 + 1 年 Cliff

### 2. 早期團隊

- 前 10 個員工決定公司文化
- 找「全能型」的人，不是「專才」
- 初期不需要完美履歷，需要能動手做事 + 認同使命的人

### 3. 執行力 > 想法

- 點子不值錢，執行才值錢
- 速度是初創公司唯一的優勢
- 每週設定 2-3 個可交付成果，週五 demo
- 不要花太多時間在完美計畫，80% 確定就開始做

## 七、融資與資源（Funding & Resources）

### 1. 融資階段

| 階段 | 金額（概略） | 來源 | 用途 |
|---|---|---|---|
| Pre-seed | < $500K | 自己、親友、天使 | 驗證想法、做 MVP |
| Seed | $500K - $2M | 天使、早期 VC | 找到 PMF |
| Series A | $2M - $15M | VC | 規模化成長 |

### 2. 不一定要融資

- Bootstrapping（自力更生）的好處：保有控制權、逼自己賺錢
- 有收入 > 有融資
- 融資是手段不是目的，不要為了融資而融資

### 3. Pitch Deck 結構（10-12 頁）

1. **問題**：痛點是什麼？
2. **解決方案**：你怎麼解決？
3. **Demo**：產品截圖或影片
4. **市場規模**：TAM / SAM / SOM
5. **商業模式**：怎麼賺錢？
6. **Traction**：目前成果（用戶數、營收、成長率）
7. **競爭分析**：為什麼你會贏？
8. **團隊**：為什麼是你們？
9. **財務預測**：未來 3 年
10. **Ask**：需要多少錢？用在哪？

## 八、成長策略（Growth）

### 1. PMF（Product-Market Fit）

- 判斷標準：「如果明天這個產品消失，你會非常失望嗎？」→ 40%+ 回答「是」
- 有 PMF 之前不要花錢投廣告
- PMF 的感覺：用戶主動推薦、需求超過你能供給的

### 2. 成長飛輪

- **內容行銷**：寫部落格、做 SEO、經營社群 → 長期免費流量
- **口碑推薦**：做好產品本身 → 用戶自願分享
- **推薦機制**：邀請好友雙方都得獎勵（Dropbox 模式）
- **社群經營**：Discord / Slack 社群 → 高黏著度
- **合作夥伴**：互相導流、聯名活動

### 3. 成長黑客思維

- 每週跑一個實驗（A/B Test）
- 追蹤 AARRR 漏斗：Acquisition → Activation → Retention → Revenue → Referral
- 找到你的「Aha Moment」（用戶體驗到核心價值的瞬間），讓更多人更快到達

## 九、創業心態（Mindset）

### 1. 必備心理素質

- **韌性**：會失敗很多次，重點是能爬起來
- **耐心**：成功通常比你想的慢，比你怕的快
- **靈活**：計畫一定會變，擁抱不確定性
- **好奇**：持續學習，對一切保持好奇

### 2. 常見錯誤

| 錯誤 | 正確做法 |
|---|---|
| 花太久做產品才上線 | 丟臉的 V1 比完美的 V∞ 有用 |
| 不跟用戶說話 | 每週至少跟 3 個用戶聊天 |
| 試圖服務所有人 | 先服務一小群人，做到極致 |
| 只關注產品不關注分銷 | 做得好 ≠ 賣得好，分銷同樣重要 |
| 太早招人 | 能不招就不招，先用外包和工具 |
| 太早優化 | 先確認方向對，再優化效率 |

### 3. 推薦資源

- 書：《精實創業》、《從 0 到 1》、《The Mom Test》、《原子習慣》
- 課程：Y Combinator Startup School（免費）
- Podcast：How I Built This、Indie Hackers
- 社群：Indie Hackers、Hacker News、Product Hunt

## 十、創業 SOP 總流程

```
1. 找問題     → 觀察痛點、記錄靈感
2. 驗證需求   → 用戶訪談、競品分析、付費測試
3. 做 MVP     → 2-4 週，只做核心功能
4. 上線測試   → 收集回饋、追蹤指標
5. 迭代改進   → Build → Measure → Learn 循環
6. 找到 PMF   → 用戶離不開你的產品
7. 規模化     → 成長策略、組建團隊、考慮融資
```

> 最重要的一句話：**先開始，再完美。**
